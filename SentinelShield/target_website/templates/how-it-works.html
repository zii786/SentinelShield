How SentinelShield Works

Your real-time digital security companion for public and government websites

Client Traffic Comes In

All user requests to the public website first pass through SentinelShield's NGINX proxy server. This allows inspection and control of all incoming traffic.

Detection Layer Kicks In

SentinelShield uses rule-based or ML-powered logic to analyze patterns. It detects anomalies like:

  • Excessive requests from one IP
  • Suspicious headers or user-agents
  • Unusual spikes in traffic

Tracer Identifies Source

When threats are detected, SentinelShield traces their origin using:

  • IP geolocation
  • Browser fingerprinting
  • Header and traffic pattern analysis

No personal user data is misused or stored.

Dashboard Displays Everything

Admins get a live dashboard showing:

  • Spikes in traffic
  • Blocked or flagged IPs
  • Attack target URLs or endpoints

Optional Alerts Triggered

If severe attacks are detected, alerts are sent to designated teams via:

  • Email (SMTP/SendGrid)
  • SMS (Twilio)
